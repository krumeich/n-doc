\moduleprocesses{prc.vpn.core}

\moduleprocess{prc.vpn.core.connect}{Open Connection to VPN concentrator}

This process opens a connection to the VPN concentrator using IPSec and
IKEv2. During the connection process, the peer presents its identity in the form
of an X.509 certificate which must be verified. It is verified using the
functionality reached by the interface \tdslink[fq]{int.vpn.cert.checkcert}.

\implementedsfr{\sfrlink{ftp_itc.1/vpn} & \sfrlink{fcs_ckm.2/ike}}

\moduleprocess{prc.vpn.core.disconnect}{Close Connection to VPN concentrator}

This process opens a connection to the VPN concentrator.

\implementedsfr{\sfrlink{ftp_itc.1/vpn}}


\moduleinterfaces{int.vpn.core}

\moduleinterfaceprovided{int.vpn.core.connect}

This interface triggers the creation of a new VPN connection (see \sectref{prc.vpn.core.connect}).

\moduleinterfaceprovided{int.vpn.core.disconnect}

This interface closes the VPN connection (see \sectref{prc.vpn.core.disconnect}).

\moduleinterfacerequired{int.vpn.cert.checkcert}

The interface \tdslink[fq]{int.vpn.cert.checkcert} is required for checking
validity of the certificate of the VPN concentrator.

\moduleinterfacerequired{int.cryptsystem.keymgmt.createkey}

The interface \tdslink[fq]{int.cryptsystem.keymgmt.createkey} is required for
creating an ephemeral key pair for the connection.

\moduleinterfacerequired{int.cryptsystem.keymgmt.destroykey}

The interface \tdslink[fq]{int.cryptsystem.keymgmt.destroykey} is required for
destroying the connection keys.
